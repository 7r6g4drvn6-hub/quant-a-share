# A股量化研究台

本项目是一个本地运行的 A 股量化 MVP，包含公开行情抓取、本地缓存、趋势动量策略、盘中实时选股、明日涨跌概率预测、日频回测和 Streamlit 页面。

## 本地启动

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
streamlit run quant_app/app.py
```

## 部署分享

推荐使用 Streamlit Community Cloud：

1. 将本项目推送到 GitHub 仓库
2. 打开 https://share.streamlit.io/
3. 选择该 GitHub 仓库
4. Main file path 填写 `quant_app/app.py`
5. 点击 Deploy

部署后会得到一个公开网址，别人可以直接打开使用。首次访问时会从公开行情源拉取股票列表；日线缓存为空时，部分回测和日线选股结果需要点击页面里的“更新日线样本”生成。

## 当前策略

市场页：

- 上证指数、深证成指、创业板指、沪深300 使用实时行情接口
- 指数表显示最新价、涨跌额、涨跌幅、开高低、成交额和行情时间
- 自动刷新时只局部更新指数区域，不整页刷新
- 新增行情预测：按指数历史相似趋势、动量、量能状态统计次日上涨概率
- 新增板块行情判断：按板块统计成交额加权涨跌幅、上涨家数占比、成交额占比、领涨股和领跌股
- 未配置 Tushare 行业时，板块先按主板、创业板、科创板等市场分组；配置 Tushare 后可用行业字段

股票数量：

- 默认股票池为成交额靠前 800 只，侧边栏最多可调到 1500 只
- 侧边栏新增“日线更新数量”，默认更新成交额靠前 300 只，用于回测、日线选股、风控和个股K线
- 侧边栏新增“实时扫描数量”，默认扫描成交额靠前 300 只，最多 800 只
- 股票池只做大范围排序；日线更新数量和实时扫描数量分别控制较重的日线请求和较轻的盘中行情请求
- 回测、日线选股、个股预测依赖本地日线缓存；样本扩大后请点击“更新日线样本”

股票池过滤：

- 排除 ST、退市标记、缺少行情的股票
- 默认取成交额最高的 1000 只
- 过滤上市天数不足和 20 日均成交额不足的股票

买入条件：

- 收盘价高于短均线
- 短均线高于长均线
- 动量排名靠前

盘中实时选股：

- 批量拉取当前价、涨跌幅、成交额、最高/最低价、报价时间
- 保留日线多头趋势：当前价高于短均线、短均线高于长均线
- 过滤盘中成交额不足、20 日均成交额不足、涨幅不在设定区间的股票
- 评分由日线动量、盘中涨幅、放量倍数、偏离短均线共同组成
- 页面可设置自动刷新间隔，刷新时只更新实时/预测区域，不整页闪烁
- “选股”页默认先看盘中实时选股；日线选股放在同页第二个子页，只在更新日线样本后变化

明日涨跌预测：

- 不是确定性预测，而是上涨概率信号
- 先统计同一股票历史上相似趋势、动量和成交额状态下的次日上涨比例
- 再用今天盘中涨跌幅、放量方向和均线状态做修正
- 输出明日方向、上涨概率、置信度、样本数和原因

最优买卖计划：

- 基于实时行情、明日上涨概率、期望收益、止损风险和收益风险比生成操作计划
- 可输入账户市值、可用现金、保留现金比例、单票仓位上限和现有持仓
- 持仓格式支持 `代码,股数,成本价,买入日期`
- 增加涨跌停、停牌/无成交和 T+1 锁定约束
- 输出买入/卖出股数、买卖金额、目标仓位、交易状态、可卖股数、期望收益额、最大风险额、止损和止盈参考
- 资金分配受现金、保留现金、单票上限、A 股 100 股一手和 T+1 约束，不会自动下单

风控和因子：

- 增加风险分组暴露，优先使用 Tushare 行业字段；没有行业时按上证主板、深证主板、创业板、科创板等市场分组
- 增加动量、趋势强度、流动性、低波动、估值和综合因子
- 选股页支持分组中性约束，避免候选股过度集中在单一分组
- 风控页展示股票池因子、回测组合分组暴露和因子暴露

参数扫描：

- 扫描短均线、长均线、动量窗口和持仓数量
- 输出累计收益、年化收益、年化波动、夏普、最大回撤、胜率和平均换手
- 用于检查参数稳定性，不建议只选历史最优参数

Tushare 可选数据源：

- 默认仍使用公开行情源，保证不配置 token 也能运行
- 配置 `TUSHARE_TOKEN` 后，会用 Tushare 补充行业、地区和市场类型等元数据
- 本地可设置环境变量：`export TUSHARE_TOKEN=你的token`
- Streamlit Cloud 可在应用 Settings -> Secrets 添加：

```toml
TUSHARE_TOKEN = "你的token"
```

回测假设：

- 使用前复权日线
- 日频调仓，等权持有
- 使用统一交易成本 bps 扣减
- 不连接券商，不执行真实交易
